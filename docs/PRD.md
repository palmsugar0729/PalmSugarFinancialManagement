# PRD — PalmSugar Financial Management

产品需求文档，基于 need_1.0.md 及 2026-06-05 需求讨论整理。

---

## 1. 产品定位

一款轻量级个人财务管理 App，支持收支记录、账单导入、理财交易管理。先做 Android，再做 iOS。

## 2. 目标用户

- 有记账需求的个人用户
- 有基金/股票投资，需要统一管理理财交易的用户
- 希望从支付宝/微信等支付工具导入账单的用户

## 3. 技术方案

| 项目 | 决策 |
|---|---|
| 框架 | Flutter（跨平台，Android 优先） |
| 数据存储 | 纯本地 SQLite，暂不上云 |
| 数据格式 | 参考用户已有的 `收支管理记录.xlsx` 结构 |

## 4. 功能列表

### 4.1 核心功能

| 功能 | 描述 | 优先级 |
|---|---|---|
| 手动记账 | 录入收入/支出，选择分类（一级+二级） | P0 |
| 分类管理 | 预设分类 + 用户自定义分类 | P0 |
| 账单文件导入 | 导入 CSV/Excel 账单文件，自动解析录入 | P1 |
| 支付宝/微信导入 | 支持支付宝和微信的账单格式解析 | P1 |
| 理财交易记录 | 手动记录基金/股票买卖 | P1 |
| 理财 App 导入 | 导入同花顺等理财 App 的交易数据 | P2 |
| Excel 导出 | 导出为 Excel 存档 | P1 |

### 4.2 其他功能

| 功能 | 描述 | 优先级 |
|---|---|---|
| 轻量化 | 安装包尽可能小，界面简洁 | 持续目标 |
| 答案之书 | 问一个问题，随机返回一句回答（内置答案库，离线） | P0（开发成本低） |
| 收支分析 | 月度汇总、分类占比、趋势图 | P1 |
| 投资交易分析 | 持仓概览、收益率计算、盈亏统计 | P1 |
| 自定义图标 | App 图标可自定义更换 | P2 |
| AI 分析 | 智能分析建议（**暂缓开发**，待讨论） | P3 |

### 4.3 基础功能 vs 高级功能

**待定**。先开发全部功能，找人试用后根据反馈区分基础/高级。

## 5. MVP 分期

### 第一期（当前）🚧

- 手动收支录入 + 分类选择
- 分类管理体系
- 当月收支汇总
- 答案之书

### 第二期

- 支付宝/微信账单导入解析
- 理财交易管理（基金买卖录入、持仓概览、盈亏计算）
- Excel 导出

### 第三期

- 深度分析图表（月度趋势、分类占比、投资曲线）
- 自定义 App 图标
- 付费功能体系搭建
- 同花顺等理财 App 数据导入

## 6. 数据结构参考

用户提供了 `收支管理记录.xlsx`，包含 5 个 Sheet：

| Sheet | 内容 | 关键字段 |
|---|---|---|
| 资产概览 | 基金持仓总览 | 基金名称、代码、类型、持有份额、当前净值、持仓金额、成本金额、收益、收益率 |
| 交易记录 | 基金买卖明细 | 日期、基金名称、代码、金额、净值、份额；分红记录（基金名称、金额） |
| 收支记录 | 日常收支流水 | 日期、类型（支出/收入/转账）、一级分类、二级分类、金额、备注 |
| 月度分析 | 月度汇总 | 月份、收入、支出、各分类占比 |
| 分类表 | 支出分类体系 | 一级分类、二级分类 |

> **成本计算采用移动加权平均法**：单位成本 = 总买入金额 ÷ 总买入份额，成本金额 = 单位成本 × 当前持有份额。

## 7. 关键决策记录

| 日期 | 决策 | 原因 |
|---|---|---|
| 2026-06-05 | 选择 Flutter 而非原生开发 | 跨平台减少维护成本，用户边做边学 |
| 2026-06-05 | 纯本地存储，不上云 | 当前无服务器预算，隐私更安全 |
| 2026-06-05 | AI 分析暂缓 | 一次性付费模式下 token 成本不可控 |
| 2026-06-05 | 基础/高级功能划分延后 | 先出功能再根据试用反馈决定 |
