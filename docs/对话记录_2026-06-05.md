## 话题 5：Excel 公式修复

- 用户新增需求：导出 Excel 功能
- 用户发现 022497（国泰CES半导体芯片）在 6/1 卖出后，收益率显示 1303%
- **问题根因**：G 列（成本金额）公式 `=SUMIF(交易记录!C:C, B, 交易记录!D:D)` 把卖出所得直接当作成本扣减，而非按成本价等比扣减
- **修复方案**：改用移动加权平均法，新公式：
  ```
  G = IFERROR(SUMIFS(buy_amounts) / SUMIFS(buy_shares) * current_shares, 0)
  ```
- **修复结果**：022497 收益率从 1303% → 55.41%，与理财 App 数据一致
- 备份文件：`收支管理记录_backup.xlsx`

## 话题 6：数据一致性问题

- 用户问：为什么有些基金和 App 对不上？
- **原因确认**：用户录入时部分交易使用了当日汇总数据而非逐笔明细，交易笔数越多偏差越大
- 受影响的基金主要是两个 QDII（70+ 笔定投）
- 还发现分红列的 SUMIF 可能因匹配字段不一致（代码 vs 名称）而漏计
