# 理财录入表单字段顺序不合理

## 现象

MuMu 模拟器测试理财录入功能，表单字段顺序与实际操作流程不符：

1. 实际操作流程：输入手续费 → 输入净值 → 输入份额 → **自动算出金额**
2. App 当前流程：输入金额 → 输入手续费 → 输入净值 → 自动算份额

用户习惯是在基金平台看到手续费、净值、份额三个数，期望填入后自动算金额，而不是反过来。

## 修复

调整表单字段顺序：
- 买入：手续费 → 净值 → 份额 → **自动算金额**（金额 = 份额 × 净值 + 手续费）
- 卖出：手续费 → 净值 → 份额 → **自动算金额**（金额 = 份额 × 净值 - 手续费）
- 金额字段移到备注上方

**涉及文件**：`lib/pages/add_fund_transaction_page.dart`
