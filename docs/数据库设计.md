# 数据库设计 — PalmSugar Financial Management

基于 `收支管理记录.xlsx` 数据结构及 MVP 分期设计。

---

## 技术选型

- **Flutter 插件**: `sqflite` + `path_provider`
- **数据库**: SQLite 3
- **版本管理**: 使用 sqflite 的 onCreate/onUpgrade 机制

---

## 表结构

### 1. categories（收支分类表）

```sql
CREATE TABLE categories (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  parent_id INTEGER DEFAULT 0,          -- 0 表示一级分类，否则为二级分类
  name TEXT NOT NULL,                    -- 分类名称
  type TEXT NOT NULL CHECK(type IN ('expense', 'income', 'transfer')), -- 收支类型
  icon_name TEXT,                        -- Flutter icon 名称（如 'restaurant'）
  sort_order INTEGER DEFAULT 0,          -- 排序权重
  is_default INTEGER DEFAULT 0,          -- 1=预设分类，0=用户自定义
  is_deleted INTEGER DEFAULT 0,          -- 软删除标记
  created_at INTEGER,                    -- 时间戳（毫秒）
  updated_at INTEGER
);
```

**说明**:
- `parent_id = 0` 表示一级分类（如"餐饮"、"交通"）
- `parent_id > 0` 表示二级分类（如"早餐"、"午餐"）
- 第一期 MVP 先支持一级分类，二级分类预留字段

**预设分类数据**（参考 Excel 分类表）：

| 类型 | 一级分类 |
|------|---------|
| expense | 餐饮、交通、购物、居住、娱乐、医疗、教育、通讯、人情、其他 |
| income | 工资、奖金、投资收益、兼职、红包、退款、其他 |
| transfer | 转账 |

### 2. transactions（收支记录表）

```sql
CREATE TABLE transactions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  type TEXT NOT NULL CHECK(type IN ('expense', 'income', 'transfer')), -- 收支类型
  amount REAL NOT NULL,                  -- 金额（元，保留两位小数）
  category_id INTEGER NOT NULL,          -- 分类 ID
  sub_category_id INTEGER DEFAULT 0,     -- 二级分类 ID（预留）
  account_id INTEGER DEFAULT 0,          -- 账户 ID（预留，第二期）
  to_account_id INTEGER DEFAULT 0,       -- 目标账户 ID（转账时使用，预留）
  date INTEGER NOT NULL,                 -- 日期时间戳（毫秒，精确到天）
  note TEXT,                             -- 备注
  tag TEXT,                              -- 标签（逗号分隔，预留）
  is_deleted INTEGER DEFAULT 0,          -- 软删除标记
  created_at INTEGER,                    -- 创建时间戳（毫秒）
  updated_at INTEGER,
  FOREIGN KEY (category_id) REFERENCES categories(id)
);

CREATE INDEX idx_transactions_date ON transactions(date);
CREATE INDEX idx_transactions_category ON transactions(category_id);
CREATE INDEX idx_transactions_type ON transactions(type);
```

### 3. answers（答案之书题库）

```sql
CREATE TABLE answers (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  content TEXT NOT NULL,                 -- 答案内容
  category TEXT,                         -- 答案分类（如 "鼓励", "建议", "提醒"）
  is_active INTEGER DEFAULT 1            -- 是否启用
);
```

**预设答案数据**（第一期内置 20 条）：
- 鼓励类："相信自己，你正在正确的道路上。"、"每一个小进步都值得庆祝。"
- 建议类："现在可能是开始储蓄的好时机。"、"检查一下你的月度预算吧。"
- 提醒类："别忘了记录今天的开销。"、"理性消费，快乐生活。"

## 预留表（第二期/第三期开发）

### 4. accounts（账户表）

```sql
CREATE TABLE accounts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                    -- 账户名称（如"支付宝"、"微信钱包"）
  type TEXT,                             -- 账户类型（cash, bank, alipay, wechat, fund）
  balance REAL DEFAULT 0,                -- 当前余额
  icon_name TEXT,
  sort_order INTEGER DEFAULT 0,
  is_deleted INTEGER DEFAULT 0,
  created_at INTEGER,
  updated_at INTEGER
);
```

### 5. fund_holdings（基金持仓表）

```sql
CREATE TABLE fund_holdings (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  fund_name TEXT NOT NULL,               -- 基金名称
  fund_code TEXT,                        -- 基金代码
  fund_type TEXT,                        -- 基金类型
  holding_shares REAL DEFAULT 0,         -- 持有份额
  current_nav REAL,                      -- 当前净值
  holding_amount REAL,                   -- 持仓金额
  cost_amount REAL,                      -- 成本金额（移动加权平均法）
  profit REAL,                           -- 收益
  profit_rate REAL,                      -- 收益率
  is_deleted INTEGER DEFAULT 0,
  updated_at INTEGER
);
```

### 6. fund_transactions（基金交易记录表）

```sql
CREATE TABLE fund_transactions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  fund_id INTEGER,                       -- 关联 fund_holdings
  type TEXT CHECK(type IN ('buy', 'sell', 'dividend')), -- 交易类型
  date INTEGER NOT NULL,                 -- 交易日期
  amount REAL,                           -- 交易金额
  nav REAL,                              -- 净值
  shares REAL,                           -- 份额
  is_dividend INTEGER DEFAULT 0,         -- 1=分红记录
  dividend_amount REAL,                  -- 分红金额
  note TEXT,
  created_at INTEGER
);
```

## 关键查询示例

### 当月收支汇总

```sql
SELECT 
  type,
  SUM(amount) as total
FROM transactions
WHERE 
  date >= ? AND date < ?  -- 当月时间范围
  AND is_deleted = 0
GROUP BY type;
```

### 分类占比统计

```sql
SELECT 
  c.name,
  c.type,
  SUM(t.amount) as total,
  ROUND(SUM(t.amount) * 100.0 / (SELECT SUM(amount) FROM transactions WHERE type = ? AND date >= ? AND date < ? AND is_deleted = 0), 2) as percentage
FROM transactions t
JOIN categories c ON t.category_id = c.id
WHERE 
  t.type = ?
  AND t.date >= ? AND t.date < ?
  AND t.is_deleted = 0
GROUP BY c.id
ORDER BY total DESC;
```

### 随机获取一条答案

```sql
SELECT content FROM answers 
WHERE is_active = 1 
ORDER BY RANDOM() 
LIMIT 1;
```
