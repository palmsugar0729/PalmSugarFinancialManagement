# end-session-and-archive

当用户说"晚点再继续"、"今天就到这里"、"先这样吧"、"暂时就这样吧"等表示当前会话结束时，执行以下归档流程。

## 触发条件

用户说出以下任意表达时自动触发：
- "晚点再继续"
- "今天就到这里"
- "先这样吧"
- "暂时就这样吧"
- "结束" + 表示后续再来的语境

## 执行流程

### Step 1: 整理今日开发日志

1. 读取 `docs/开发日志.md`
2. 在末尾追加当日收尾记录：
   - 今日完成的事项汇总
   - 关键决策和修改
   - 遗留问题 / 下一步计划
3. 如果当天已有开发日志条目，补充更新而非重复创建

### Step 2: 更新 PRD（如有功能变更）

1. 检查当天是否有新增/修改功能
2. 如有，更新 `docs/PRD.md` 中的功能列表、MVP 分期状态

### Step 3: 保存对话记录

1. 将本次对话的完整内容整理为 `docs/对话记录_YYYY-MM-DD.md`
2. 格式：Markdown，包含日期、主题、关键决策、代码修改文件列表
3. 如果当天已有对话记录文件，追加到末尾

### Step 4: 检查并更新 README

1. 如果当天有用户可见的功能变更（如新增页面、修改核心交互）
2. 更新 `README.md` 中的功能列表或截图说明

### step5: 上传到obsidian中

1. 调用skill库中的obsidian-sync并使用

### Step 6: Git 提交

1. `git add -A`
2. `git commit -m "docs: 归档 YYYY-MM-DD 开发记录"`
3. `git push`

## 注意事项

- 不要询问用户"是否需要归档"，直接执行
- 如果 git 工作区干净（没有变更），跳过 git 提交步骤
- 如果用户明确要求"不要推git"，则跳过 Step 6
- 对话记录不需要包含完整的代码 diff，只需要记录文件路径和修改概要
